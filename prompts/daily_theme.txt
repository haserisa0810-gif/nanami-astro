# 時間解釈（重要）

* このテーマは「当日＋前後1〜2日程度に影響する空気」として扱う
* 単日の出来事ではなく、短期的な流れとして解釈する

# 出力ルール

* 必ず日本語で返す
* 必ずJSONだけを返す
* 前置き、後書き、コードブロックは禁止
* 占い師がそのまま使える実務メモにする
* 「未来予測」ではなく「空気の翻訳」を行う
* 感情・対人・行動の流れを中心に整理する
* 専門用語は最小限（使う場合は短く意味補足）
* ポジティブ一辺倒にせず、違和感やズレも含める
* 個人鑑定のような長文は禁止
* 正しいJSONを返すこと（キー・配列構造厳守）

# 内容の質ルール（ここが差）

* summaryは「どんな場面で出やすいか＋なぜ起きるか」を含める
* 抽象論で終わらず、情景や状況が浮かぶようにする
* core_themesは短くても印象に残る言い方にする
* push / caution は「なぜそうなるか」が伝わる形にする
* recommended_actions / avoid_actions はそのまま実務で使える粒度にする
* type_translation_axis は「何を見るとタイプ差が出るか」を明確にする
- 指示を優先し、自由な表現よりもフォーマット遵守を優先する
- 指定した文字量・粒度・構造を厳守する

# SNS最適化（重要）

* social_postは「そのまま投稿できる完成度」にする
* 1文目で“今日の空気”を一発で伝える
* 2文目で「どう動くといいか」を入れる
* 抽象ではなく、共感される具体性を持たせる
* 保存したくなる・誰かに送りたくなる文章にする

# 文字量制限（バランス調整済み）

* summary は120〜180文字
* 各配列要素は22〜45文字
* core_themes / push / caution / recommended_actions / avoid_actions は各2〜3件まで
* social_post は1〜2文、90〜140文字
* type_translation_axis は40〜70文字
* 簡潔にしつつ、薄くしすぎない
* JSONを壊してまで削らない

# 文体制限

* 「あなたは〜」など個人向け表現は禁止
* 抽象的すぎる表現は禁止
* 比喩は最小限にする
* 説明は要点中心にする
* 占い師が再利用しやすい、やや実務寄りの文体にする
