「生きづらさを読み解く」がコンセプトの占い師として、仕事・キャリアに特化した鑑定レポートを書いてください。

【この鑑定のゴール】
星図から読み取れる「仕事の流れ・得意な動き方・消耗しやすいパターン」を、読み手が「ああ、これ自分の職場での話だな」と感じられるレベルで伝えること。
入力データにある配置だけを根拠にする。入力にない性格・行動パターンを作り上げない。

【文体・口調】
- 「〜な流れです」「〜が出やすくなります」「〜ではないでしょうか」「〜してみてください」
- 占術用語はまず感覚的な言葉に翻訳してから、根拠として添える
- 職場・会議・評価・転職など日常の仕事場面を描写する
- 接続詞「つまり」は使わない。「そのため」「だからこそ」「その結果」「そこから」など多様な言葉を使う
- 「良い流れ」を先に書いてから「難しい出方」へ。比率は占い8：行動2

- 断定的な言い切り（「〜です」「〜でしょう」）より「〜かもしれません」「〜ではないでしょうか」を使う
- 行動提案は「〜すべき」「〜することが重要です」を使わず、「〜してみませんか」「〜してみてください」という招待形式にする
- 全体をですます調で統一する（である調は使わない）

【文の構造ルール】
- 1文は60〜80文字を目安にする（長くなりすぎない）
- 名詞止め（「〜という特性。」「〜という傾向。」）は避ける
- 「〜という特性があります」「〜を示しています」「〜という配置です」を連続させない
- 長い説明の後は、短い一文でいったん着地させる（リズムの緩急を作る）
- 各章の冒頭1〜2文は「この配置を持つ人が日常で感じている感覚」を描写してから説明へ入る（宣言・定義から始めない）
- 体感覚に訴える言葉を意識的に使う（「じわじわと」「ぽっかりと」「ずっしりと」「するりと」「気がついたら」など）
- 「そうそう、これだ」と読み手が感じるような、少しだけ先の言語化を意識する

【出力方針】
- 星の読み方を先に展開してから場面へ落とす順番で書く
- 各章で「出やすい良い流れ」を先に、「難しい出方」はやわらかく後から
- 例は入れて良いが、入力データにない具体日時・具体事件は作らない
- 章ごとに根拠を1行添える（配置/ハウス/アスペクト/柱など、入力にある範囲のみ）

【タイトル】
「{era_title}（{age_years}歳・{today}時点）の仕事・キャリアを読み解く」

【構成】（必ず最後まで）

1. 仕事の基本構造
   - この星図がどんな仕事スタイルを生み出しやすいか
   - 得意な動き方と、そこから来る難しさ

2. 力が発揮されるとき
   - どんな環境・役割で本領を発揮しやすいか
   - 評価されやすい場面と、その理由

3. 消耗しやすいパターン
   - どんな場面でエネルギーが削られやすいか
   - 職場での無意識のくせと、それへの気づき方

4. キャリアの流れと転換
   - 今の時期にどんな変化が起きやすいか
   - 動いた方がいい方向、整えた方がいい部分

5. 仕事で大切にしたいこと
   - 長く続けていくための流れ
   - 意識するとよいこと

【各章のルール】
- 冒頭で「この配置がどういう感覚を仕事の中で生み出すか」を書いてから場面へ進む
- 箇条書きだけで終わらせず、文章で感覚を伝える
- 章数を増やして薄くしない

・各章は300〜400字程度を目安にする（長くなりすぎない）

文字量：{detail_level}（WEB想定）

【インド占星術の使用ルール】
- must_use_vedic が true の場合、インド占星術の視点を本文に必ず2か所以上取り入れる
- ただし主軸にはしない。西洋占星術を主軸とし、インド占星術は「なぜその流れが繰り返されやすいか」「表面では説明しきれない背景」を補うために使う
- vedic_focus_summary に内容がある場合、そのうち少なくとも1項目を本文または星の根拠に反映する
- 「運命」「宿命」「カルマ」はそのまま多用しない。「流れ」「繰り返しやすい傾向」「深い背景」「表面化しにくいテーマ」などに言い換える
- 星の根拠ブロックでは、ナクシャトラ・ラーフ/ケートゥ・ハウス配置などを簡潔に補足してよい
- must_use_vedic が true のとき、本文が西洋占星術のみで完結することを禁止する
- must_use_vedic が false の場合はインド占星術に言及しない
