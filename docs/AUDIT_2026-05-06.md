# nanami-astro 機能棚卸しレポート

作成日: 2026-05-06

## 結論

現時点で「すぐ消すべき危険機能」は見つかっていません。ただし、公開API・公開URL・ファイルアップロード・生成AI・外部通知が増えているため、運用上の事故リスクは上がっています。

優先度が高い見直しは次の4つです。

1. 公開APIにレート制限または簡易APIキーを入れる
2. アップロード系のサイズ・拡張子・保存先・HTML安全性を統一する
3. 生成ファイルの保存期間と自動削除ルールを決める
4. 試作/旧版サービスを `experimental` または `legacy` として明示する

## ルート構成

`routes.py` で有効化されている router:

- `line_webhook`
- `routes_public_pages`
- `routes_api_calc`
- `routes_analyze`
- `routes_analyze_save`
- `routes_transit`
- `routes_public_orders`
- `routes_reader`
- `routes_admin`
- `routes_staff`
- `routes_external_orders`
- `routes_daily_theme`
- `routes_daily_card`

管理系は大きく分けて `admin`、`staff`、`reader`、`products` に分かれています。公開導線は `public_pages`、`public_orders`、`daily_card`、`daily_theme`、`api_calc`、`transit` です。

## 認証まわり

スタッフ/管理/鑑定士向けの主要画面は概ね `Depends(get_current_staff)`、`Depends(get_current_admin)`、`Depends(get_current_reader)` が入っています。

確認できた認証付きの代表例:

- `/staff/orders...`
- `/staff/external-orders...`
- `/products/transit...`
- `/products/social-scenarios...`
- `/admin/...`
- `/reader/...`

公開URLとして意図的に認証なし:

- `/`
- `/western`, `/vedic`, `/integrated`, `/guide`, `/about`
- `/order/...`
- `/free-reading/...`
- `/result/{order_code}`
- `/report/{order_code}`
- `/report/share/{token}`
- `/t/{token}`
- `/daily-card...`
- `/daily-theme...`
- `/api/calc/...`
- `/transit/...`

注意:

- `/report/{order_code}` と `/result/{order_code}` は注文番号が実質アクセスキーです。注文番号が推測しやすい形式なら、トークン化を検討してください。
- `/report/share/{token}` と `/t/{token}` はトークン付き公開URLなので設計としては妥当ですが、有効期限・再発行・無効化の運用確認が必要です。
- `/api/calc/*` と `/transit/*` は計算資源を使う公開APIです。悪用や高負荷対策として、レート制限またはAPIキーを検討してください。

## 外部API・秘密情報

利用箇所:

- Anthropic Claude: `services/report_generation_service.py`, `services/social_daily_scenario.py`
- Gemini: `routes_daily_theme.py`
- Google Cloud Storage: `routes_staff.py`, `routes_external_orders.py`, `services/report_generation_service.py`, `services/external_pdf_service.py`
- LINE: `services/notification_service.py`, `services/daily_card_service.py`, webhook系
- Stripe: `routes_public_orders.py`, models
- VOICEVOX: `services/social_daily_scenario.py`
- TikTok/Instagram投稿API: `services/social_daily_scenario.py`

リスク:

- 生成AIに渡すプロンプトには個人情報が含まれます。ログ出力・保存先・閲覧権限の棚卸しが必要です。
- `services/social_daily_scenario.py` のSNS投稿関数は dry-run デフォルトなので現状は安全寄り。ただし本番投稿を有効化する場合は、画面上の明示確認と権限分離が必要です。
- FreeAstro は整理済み。旧フォールバック導線、専用サービス、`FREEASTRO_*` 設定例は削除しました。

## ファイルアップロード・配信

アップロード/配信の代表箇所:

- `/staff/orders/{order_code}/upload-report-html`
- `/staff/external-orders/{order_id}/upload-html`
- `/products/transit/{request_id}/html`
- `/api/products/transit/{request_id}/html`
- `/products/social-scenarios/upload`
- `/products/social-scenarios/files/{filename}`
- GCSへのHTML/PDF保存

良い点:

- SNS素材画面はスタッフ認証付き。
- SNS素材のファイル配信もスタッフ認証付き。
- SNS素材アップロードは拡張子制限あり。
- 外部受注HTMLアップロードは5MB制限あり。

改善候補:

- SNS素材アップロードにもファイルサイズ上限を追加する。
- HTMLアップロード系はスクリプト混入リスクがあるため、公開前にサニタイズまたは「信頼済みスタッフのみ」の運用明記が必要。
- GCS公開URLとDB公開URLのライフサイクルを整理する。
- `outputs/social_scenarios` など生成ファイルの自動削除ルールを追加する。

## 公開API・高負荷リスク

公開API:

- `/api/calc/western`
- `/api/calc/shichu`
- `/api/calc/transit`
- `/api/calc/combined`
- `/transit/single`
- `/transit/synastry`
- `/transit/interpret`
- `/daily-theme/generate`
- `/daily-theme/generate-types`

特に注意:

- `/daily-theme/generate` と `/daily-theme/generate-types` はGeminiを呼び得るため、外部から連打されるとコストが発生します。
- `/api/calc/combined` は計算量が比較的大きい可能性があります。
- `/transit/interpret` が生成AIや重い処理に接続している場合は、公開のままにするか再判断が必要です。

推奨:

- 公開APIにIPベースの簡易レート制限を追加。
- 生成AIを呼ぶ公開APIは `X-Internal-Token` かCSRF相当の保護を検討。
- 失敗時に例外文字列をそのまま返す箇所は、内部情報を出しすぎないように整える。

## 生成AI機能

主な機能:

- 通常鑑定AI生成
- 外部受注HTML鑑定書生成
- 今日の空気生成
- SNS日次シナリオ生成

リスク:

- プロンプトが多く、どれが本番導線か把握しにくい。
- モデル名が複数箇所に分散している。
- 最大トークンや再試行の方針が機能ごとに違う。
- 個人情報を含むYAML/HTML/ログが残りやすい。

推奨:

- `prompts/README.md` を作り、各promptの用途・呼び出し元・現役/旧版を明記。
- モデル名・最大トークン・タイムアウトを設定サービスに集約。
- AI生成ログの保存期間を決める。

- 大きなテンプレートは「新規機能を足さない」ルールにして、追加は別テンプレートへ。

## 最近追加したSNS素材機能

現状:

- `/products/social-scenarios` はスタッフ認証付き。
- 投稿APIは実装済みだが、CLIで `--live-post` を付けない限り dry-run。
- Web画面は投稿前素材の確認・アップロード用途で、自動投稿はしない。

良い点:

- 投稿担当者が仕上がり確認できる。
- 素材配信がスタッフ認証付き。
- 投稿自動化を急いでいないため、誤投稿リスクは低い。

追加した方がよいもの:

- アップロードサイズ上限。
- 古い素材の削除ボタンまたは自動削除。
- mp4自動生成を入れる場合は `ffmpeg` の存在確認とエラー表示。

## すぐやる推奨タスク

1. `/daily-theme/generate*` に簡易保護を入れる  
   公開画面から使うならレート制限、内部生成ならトークン必須。

2. `/api/calc/*` の公開方針を決める  
   外部APIとして売る予定ならAPIキー、内部用なら認証を追加。

3. SNS素材アップロードにサイズ上限を追加  
   例: 100MBまで。動画を扱うなら設定値化。

4. 生成ファイルの掃除  
   `outputs/`, draft, report, social assets の保持期間を決める。

5. 旧版候補のラベル付け  
   削除ではなく、まず `legacy` / `experimental` を明記する。

## メモ

このレポートでは、FreeAstroの旧フォールバックのみ削除済みです。その他の項目は、現役機能・公開面・危険面・整理候補を分けるための棚卸しです。
